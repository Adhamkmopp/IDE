Show of hands

The first dataset used for the following visualisations is a 112-dimensional dataset with 40 data points in it. Each data point represents the outline of a hand. The first 56 variables are x-coordinates of the points around the hands outline and the next 56 variables are y-coordinates. The second dataset used is the result of applying the PCA (Principal Component Analysis) algorithm. PCA is used for dimensionality reduction of high-features datasets. It is defined as a linear transformation that transforms the data to a new coordinate system such that the greatest variance by some projection of the data comes to lie on the first coordinate. The original dataset provided has been modified using this script to add a header to the columns. The dataset contains 40 rows, one for each hand and 112 columns one for each of the principal components.

The plot on the right shows the first two PC from the PCA data set. The points have been clustered with the k-means algorithm using the library clusterfck. The number of clusters set is 5 and the points are coloured accordingly. Note that the k-means algorithm initialises random centroids every time it is run. This leads to different clusters and colorings every time that the page is being reloaded. The randomness leads sometimes to show the presence of outliers (hand 37 and hand 39).


The toggle button allows the user to enter two modalities:

  • One hand outlines: When the toggle button is grey then the user can clicks on a point in the PCA plot and the outline of that hand is shown in the visualisation on the left.
  • All hand outlines: When the toggle button is clicked and it turns blue, all the outlines of all the hands are shown together. Furthermore, when the user hovers over a point in the PCA plot, the corresponding outline is highlighted in the visualisation on the left.

In addition there is a drop-down menu that allows the user to choose which two PC are plotted against each other.

Looking at the PCA plot we see in general that points that are closer together have a similar outline. It is possible to notice that the first PC component catches the hands thickness while second PC catches the separation between the fingers. On top of that, starting from PC 39 up to 112 the values of the variance captures values of zero.

In conclusion, it is apparent that the first two PCs are enough to capture the variance of the shape of the hands, or at least the hand thickness and separation of the fingers.
